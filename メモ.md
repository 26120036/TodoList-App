## CTRL + Cの検出
[書かれていたQiita](https://qiita.com/tas6/items/5da454dac31a11a9c2b5)  
RuntimeのaddShutdownHookメソッドで登録する。 
また、このままだとCTRL + Cすると`java.util.NoSuchElementException`が出てくる。
try-catch で受け取り空文字を返すようにした。  
```java
Runtime.getRuntime().addShutdownHook(new Thread(() -> {
    // この中にメッセージや終了処理をかく
}));
```

## todos(ArrayList)の空確認
今回は最初に`todos = new ArrayList<>();`で初期化をしているため、`== null`するとエラーが出た。  
ArrayListの空確認は`todos.isEmpty()`で行う。

## ファイルの場所、文字コード指定、ファイル保存じの改行
文字コードUTF-8で明示。  
`FileWriter writer = new FileWriter("practice/todo.txt", StandardCharsets.UTF_8)`  
  
最後に\nをつけて改行を入れておく。  
`writer.write(title + "," + status + "\n");`

## ファイルを1行ずつ読み込む
FileReaderは1文字ずつ読み込むため、今回は使わない。  
BufferedReaderが持っている`readLine()`を使って1行ずつの読み込みができる。  
[readLine()に関する記事](https://www.sejuku.net/blog/20924#index_id3)  
`BufferedReader reader = Files.newBufferedReader(path, StandardCharsets.UTF_8);`  
`line = reader.readLine()`

## カンマで分割
今回は`title,boolean`の形式で保存されているので、`split()`を使ってカンマで区切り、配列にして渡す。  
`private static final String DELIMITER = ",";`で区切る文字を定数にしておく。  
`String[] data = line.split(DELIMITER);`

## 起動時のファイル読み込み
最初に起動した時はtodo.txt内のデータがtodos(arrayList)にaddされていないので初期化されたままになっている。  
なのでMainファイルで入力開始する前に`todoManager.load()`を行っておく。
